## Lite Must-Keep Guards

- [ ] User-only acceptance remains required.
- [ ] expectedFiles are known.
- [ ] Minimal AC is concrete.
- [ ] Minimal Test/Evidence plan exists.
- [ ] File Change Guard can explain source changes.
- [ ] Product meaning does not change.
- [ ] No direct pbe-state edit.

## Escalation Checks

- [ ] Product meaning changes.
- [ ] AC is still ambiguous.
- [ ] UI/UX taste or visual direction is involved.
- [ ] DB/schema/auth/permission changes.
- [ ] External API/hardware/concurrency involved.
- [ ] Multiple modules or broad expectedFiles.
- [ ] Product Patch required.
- [ ] Repeated review rejection.
- [ ] High-risk manual-only evidence.

## Decision

- [ ] Stay Lite
- [ ] Escalate to Full
- [ ] Bypass is acceptable
